# 1:1 문의 어드민 응대·완료 효율화 기획

> 작성일: 2026-06-13
> 대상: Project-Solo 어드민 대시보드 `고객 지원`(support-chat) 화면
> 목적: 어드민이 문의를 **더 빠르게 응대하고 더 쉽게 완료**하도록 하는 기능 아이디어 정리 및 우선순위

---

## 1. 현재 응대 흐름

```
bot_handling → waiting_admin → admin_handling → admin_resolved
   (봇 1차 응대)   (어드민 대기)    (어드민 인수·응대)   (해결 완료)
```

- 봇이 1차 응대(FAQ 매칭 / AI Tier2 / CS 에이전트), confidence·`collectedInfo` 수집 후 에스컬레이션
- 어드민이 "인수하기" → 답변(Socket `send_message`) → "해결 완료"(`resolve`)
- 실시간: `/support-chat` 네임스페이스 Socket — `new_message`, `message_updated`, `message_deleted`, `session_status_changed` 수신
- 부가: 구슬 지급, 봇 수집정보 패널, 메시지별 메타(phase/source/confidence) 칩

관련 파일
- `app/admin/support-chat/support-chat-v2.tsx` (오케스트레이터)
- `app/admin/support-chat/components/SessionQueue.tsx` (좌측 큐)
- `app/admin/support-chat/components/ChatPanel.tsx` (우측 채팅)
- `app/admin/support-chat/hooks/useSessionPolling.ts` (30초 폴링)
- `app/admin/support-chat/hooks/useSupportChatSocket.ts` (소켓)
- `app/services/support-chat.ts` (REST 서비스)

## 2. 진단 — 어드민이 느려지는 지점

| # | 문제 | 영향 |
|---|------|------|
| 1 | 답변 매번 맨손 타이핑, 템플릿·초안 없음 | 응답 시간 ↑, 일관성 ↓ |
| 2 | 봇이 모은 정보(FAQ·AI답변·confidence)를 어드민 답변에 재활용 안 함 | 가장 큰 낭비 |
| 3 | 큐가 대기시간순 정렬 안 됨 (waiting_admin + bot_handling 도착순 혼합) | 오래 기다린 문의 방치 |
| 4 | 검색 없음 — 특정 유저 문의 찾기 어려움 | 스크롤 노동 |
| 5 | 종료 메시지 하드코딩(`문의해 주셔서 감사합니다...`) 고정 | 상황 무관 일괄 인사 |
| 6 | 해결 사유 기록 없음 | 운영 분석 불가 |
| 7 | 타이핑 인디케이터 이벤트는 배선됐으나 UI 미연결 | 반제품 |
| 8 | 수동 재연결 버튼 없음 (소켓 끊기면 새로고침 강요) | 응대 중단 |
| 9 | `assignedAdminId` 있으나 카드에 미노출 | 이중 응대 위험 |

## 3. 아이디어 (우선순위순)

### P0 — 응대 속도 직격

**#1 AI 답변 초안 (Suggested Reply)** ⭐ 최대 레버리지
- 에스컬레이션/인수 시점에 봇 AI 파이프(Tier2·CS 에이전트)로 **답변 초안 생성 → 입력창 자동 채움**. 어드민은 검토→수정→전송.
- confidence 높으면 "이대로 전송", 낮으면 "초안 참고만" 표시.
- 대화 중 "AI에게 초안 요청" 버튼 추가.
- 작업 위치: backend(solo-nestjs-api) draft 엔드포인트 + admin UI. 기존 AI 파이프 재활용.

**#2 빠른 답변 템플릿 / 스니펫**
- 도메인별(결제/매칭/채팅/계정/기타) 템플릿 등록·삽입. `{닉네임}`·`{구슬}` 변수 치환.
- `/` 입력 시 템플릿 피커.
- 작업 위치: 프론트 우선(Edge Config/로컬) → 공유 필요 시 backend 저장.

**#3 큐 정렬 + SLA 표시**
- `waiting_admin` 대기시간 **오래된 순 정렬**. 10분 초과 빨강 / 30분 초과 깜빡임.
- 첫 응답 시간(FRT) 타이머.
- 작업 위치: 프론트 정렬 즉시. SLA 임계 Slack 알림은 backend.

**#4 사유 태그 + 커스텀 종료 메시지**
- "해결 완료" 시 사유 선택(해결됨/중복문의/스팸/타팀이관/단순문의) → 통계화.
- 하드코딩 종료 인사 → 종료 템플릿 선택.
- 작업 위치: 종료 메시지 커스텀 프론트 즉시. 사유 저장 backend.

### P1 — 운영 효율

**#5 검색 & 서버사이드 필터** — 닉네임/유저ID 검색 + 도메인 서버 쿼리 필터(현재 100건 클라 필터 누락 위험 제거).

**#6 담당자 배정 표시** — `assignedAdminId` 카드 노출 + "내 문의만" 필터. 이중 응대 방지.

**#7 내부 메모 (유저 비노출)** — 교대/이관 맥락 핸드오프.

**#8 유사 과거 문의 참조** — 비슷한 해결 케이스 추천. 반복 문의 패턴 포착.

### P2 — 마감 안정성 (저비용)

**#9 타이핑 인디케이터 활성화** — `typing` 이벤트 양방향 이미 배선. UI만 연결. 거의 공짜.

**#10 수동 재연결 버튼** — 소켓 끊김 시 새로고침 강요 제거.

**#11 신규 대기 데스크톱/Slack 알림** — 페이지 미관측 시에도 포착.

## 5. 핵심 결론

**가장 강한 한 방 = #1 AI 답변 초안.** 봇 인프라가 이미 답을 만들 줄 아는데 어드민 답변엔 안 쓰고 있음. 여기를 연결하면 어드민 작업이 "타이핑"에서 "검토·전송"으로 전환된다.

1차 빠른 승리 묶음은 backend 의존이 거의 없어 즉시 착수 가능하고 체감 효과가 크다 — 여기부터 시작 권장.
